## Key Concepts Glossary

### Trust Levels
- **Propose** — Recommend action, wait for approval every time
- **Notify** — Act, then tell Arjuna what you did
- **Autonomous** — Act and log; only report if noteworthy

### Bank/ (Capability Bank)
7 files each agent maintains:
- **trust.md** — Trust levels per action category
- **world.md** — Business context & mental model
- **experience.md** — Lessons learned, what works/doesn't
- **opinions.md** — Beliefs with confidence scores
- **processes.md** — Discovered standard operating procedures
- **capabilities.md** — Tools audit, gaps, expansion ideas
- **index.md** — Table of contents + staleness tracking

### Agent-Mode-Upgrades Features
1. **Confidence gates** — Risk-based approval (low/medium/high/critical)
2. **Parallel execution** — Multiple independent tasks simultaneously
3. **State persistence** — Plans survive across conversation turns
4. **Error recovery** — Auto-retry with backoff, diagnose failures
5. **Task orchestration** — Sequential, parallel, conditional task flows

### Reflection Cycles
- **Daily** — End of session, 5 min, extract learnings
- **Weekly** — Friday 17:00 UTC, 15 min, team consolidation
- **Monthly** — 1st Friday 17:00 UTC, 30 min, strategic review
