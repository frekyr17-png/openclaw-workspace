### Team Knowledge Base
```

├── README.md          # How to use
├── patterns/          # Proven patterns (trading, security, project-mgmt, operations)
├── decisions/         # Major decisions
└── weekly-digests/    # Weekly team summaries
```

### Automation
- **Daily reflection:** Added to Arjuna's HEARTBEAT.md (1x/day rotation)
- **Weekly digest:** Script at `scripts/weekly-team-digest.sh` (ready for cron)
- **Monthly compaction:** Planned (merge rules, archive old patterns)

### Learning Flow
1. **Agent works** → logs corrections, reflections, wins
2. **Pattern emerges** (3x success) → promotes to HOT tier (memory.md)
3. **Shares with team** → adds to shared-learnings/patterns/
4. **Other agents read** weekly digest → apply relevant insights
5. **Results feed back** → reinforces or demotes pattern

### Memory Tiers
- **HOT** (memory.md): ≤100 lines, always loaded, core rules
- **WARM** (projects/, domains/): Load on demand when context matches
- **COLD** (archive/): 90+ days unused, load on explicit query

### Key Features
- ✅ No manual maintenance (auto-promote/demote based on usage)
- ✅ Cross-agent knowledge sharing (Richie's trading wins → Arjuna's execution)
- ✅ Prevents repeated mistakes (log corrections → remember forever)
- ✅ Compounds over time (old knowledge archives, doesn't delete)
- ✅ Privacy-aware (personal info stays personal)
