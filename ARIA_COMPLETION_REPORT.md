## Agent-Mode-Upgrades Configuration

All 5 agents are configured for:

### ✅ Confidence Gates (Risk-Based Approval)
- **Low risk** (auto-approve): Read, analysis, research
- **Medium risk** (notify): Write, edit, safe commands
- **High risk** (require approval): Messages, browser, git push
- **Critical risk** (always ask): rm -rf, database changes, credential changes

**Per agent:**
- Arjuna: All autonomous (leader authority)
- Richie: Trades > $10 require approval
- Sonic: Outreach > 50 leads requires approval
- Bruce: Capability restrictions require approval
- Aria: Deadline changes > 1 day require approval

### ✅ Parallel Execution
- Max 5 concurrent workers per agent
- Max 15 tasks/hour across team
- Richie: 2-3 strategy backtests simultaneously
- Sonic: 5 prospects researched simultaneously
- Bruce: Multiple security scans on different systems
- Aria: 3+ parallel projects tracked

### ✅ State Persistence
- Agent plans survive across conversation turns
- Trading strategy step progress saved
- Lead gen campaign outreach status tracked
- Security audit partial results preserved
- Project task state persists across sessions

### ✅ Error Recovery (Auto-Retry)
- Diagnose error type (permission, network, not_found)
- Apply fixes (add sudo, increase timeout, backoff)
- Retry up to 3 times with exponential backoff
- Escalate if all retries fail

### ✅ Task Orchestration
- Sequential: A → B → C (step by step)
- Parallel: A & B simultaneously, then merge C
- Conditional: If X do Y, else do Z
